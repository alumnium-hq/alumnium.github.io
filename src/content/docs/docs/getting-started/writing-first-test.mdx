Let's see how we can write a simple test that would open DuckDuckGo, search for a Mercury element, ensure that search results are loaded, include the Wikipedia article about the element, and check that its symbol is presented correctly on the page.

:::note
Alumnium can be used with any test runner you like. For the examples in this guide, we'll use [Pytest][2].
:::

## Install Dependencies

Make sure you have Alumnium and either Selenium or Playwright installed in your project.

<Tabs syncKey="driver">

```bash title="Installing dependencies..."
$ pip install alumnium selenium
```

</TabItem>
<TabItem label="Playwright">

```bash title="Installing dependencies..."
$ pip install alumnium pytest-playwright
$ playwright install chromium
```

</TabItem>

</Tabs>
</LanguageContent>

## Setup Browser

Start by creating a test file for your tests and instantiating a browser instance:

<LanguageContent lang="python">

<Tabs syncKey="driver">
<TabItem label="Selenium">

```python title="test_search.py"
import unittest
from selenium.webdriver import Chrome


class TestSearch(unittest.TestCase):
    def setUp(self):
        driver = Chrome()
        driver.get("https://duckduckgo.com")

    def test_search(self):
        pass
```

</TabItem>
<TabItem label="Playwright">

```python title="test_search.py"
from playwright.sync_api import Page


def test_search(page: Page):
    page.goto("https://duckduckgo.com")
```

</TabItem>
</Tabs>

Now run the test. You should see a Chrome browser window with DuckDuckGo opened.

```bash title="Running test..."
$ pytest --quiet
...
1 passed in 1.32s
```

:::note
For Playwright, you might need to run with `--headed` flag to see the browser window.
:::

</LanguageContent>

## Add Actions

Common wisdom says to _never trust a test you haven't seen fail_. Let's add the first verification and see it fail!
